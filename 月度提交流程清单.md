# FPP 月度提交流程清单（US & CA）

> 适用范围：`工作提效` 目录下的美国/加拿大 FPP 月度提交工作  
> 建议使用方式：每月复制一份，按步骤勾选并留痕

---

## 0. 月度启动

- [ ] 确认本期期间（例如：`202606`）
- [ ] 在共享盘创建本期目录（US、CA、数据来源）
- [ ] 确认本期负责人、复核人、提交截止时间

---

## 1. 收集原始材料（证据链起点）

- [ ] 收齐当月供应商邮件（US/CA）
- [ ] 收齐发票或替代凭证（PDF）
- [ ] 收齐人员数据（HC Analysis 来源）
- [ ] 所有材料按统一命名保存

命名建议（统一模板）：

`国家_供应商_费用类型_期间_版本.后缀`  
示例：`US_Cigna_Dental_202606_v1.pdf`

---

## 2. 目录整理与完整性检查

- [ ] US 福利供应商目录存在当月子目录（`2026MM`）
- [ ] 各月目录下包含 `HC Analysis` 和发票目录（Invoice）
- [ ] CA SunLife 目录存在当月子目录（`2026MM`）
- [ ] 数据来源目录存在对应期间的人力分析文件

建议先跑自动检查脚本：

```bash
python3 check_fpp_files.py --root "/Users/zhangwenjing/Desktop/工作提效"
```

---

## 3. 分摊计算与台账更新

- [ ] 基于 HC 数据完成人数/金额分摊
- [ ] 更新供应商分摊明细（US/CA）
- [ ] 更新 `AMER FPP Monthly Report.xlsx`
- [ ] 在 `Issue Tracker` 记录异常项和处理人

---

## 4. FPP 系统填单（Payment Request）

- [ ] 进入正确场景（US/CA）
- [ ] 加载正确合同（Contract No.）
- [ ] 填写付款信息（金额、币种、描述、账期）
- [ ] 检查审批链（Requester/Reviewer/Department/Center）
- [ ] 上传发票或替代凭证
- [ ] 上传 Other Attachment（对账单、计算表等）
- [ ] 导入会计分摊模板（Batch Upload）

---

## 5. 提交前 5 项强校验

- [ ] 金额一致：发票 = 分摊结果 = 系统金额
- [ ] 币种一致：USD/CAD 与合同一致
- [ ] 期间一致：目录期间 = 文件名期间 = Payment Description 期间
- [ ] 附件齐全：邮件、发票、分摊表、说明材料
- [ ] 命名规范：无临时命名、无歧义版本

---

## 6. 提交与归档闭环

- [ ] 完成提交并记录 FPP 单号
- [ ] 将单号回填到月报和 Issue Tracker
- [ ] 将最终版资料归档到当月目录
- [ ] 给相关方发送完成通知（含单号和目录路径）

---

## 7. 复盘（可选但建议）

- [ ] 统计本月耗时（收集/计算/提单/复核）
- [ ] 记录 Top3 异常及根因
- [ ] 明确下月优化动作（标准化、脚本化、模板化）

---

## 常见问题快速定位

- 目录缺文件：先跑 `check_fpp_files.py`，根据 Missing 项补齐
- 期间不一致：优先修正文件名与目录期间，再修 Payment Description
- 审批退回：在 `Issue Tracker` 记录退回原因，沉淀为下月预检规则
