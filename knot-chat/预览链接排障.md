# Knot 预览链接打不开 · 排障手册

## 先分清：你在哪里打开？

| 打开方式 | 能不能用 |
|----------|----------|
| Mac 浏览器开 `http://127.0.0.1:18081/...` | ❌ 127.0.0.1 是 Knot 机器本机，不是你的 Mac |
| Mac 浏览器开 `https://xxx.preview.with.woa.com/upload-docs.html` | ✅ 正确方式 |
| 旧的 preview 链接（几天前复制的） | ⚠️ 可能已过期，需重新生成 |

---

## 第一步：在 Knot 工作区终端确认服务在跑

```bash
cd /data/workspace/amer-fpp-a
bash knot-chat/restart_web.sh
```

应看到两行 **HTTP 200**。若是 000 或非 200 → 看日志：

```bash
tail -30 运行日志/backend.log
tail -30 运行日志/web.log
```

常见原因：
- `address already in use` → 用 `restart_web.sh` 会先杀旧进程
- `main:app not found` → `git pull` 后缺少 `bot-gateway/main.py`
- `ModuleNotFoundError: fastapi` → `pip3 install -r bot-gateway/requirements.txt`

---

## 第二步：重新生成预览链接（Knot 集成页）

1. Knot → **智能体集成** / **工作区** → 找到 **服务预览** 或 **生成预览链接**
2. **端口选 `18081`**（不是 18082）
3. 生成后，浏览器地址必须是：

```text
https://新的预览域名/upload-docs.html
```

⚠️ 不要只打开根路径 `/`，必须带 `/upload-docs.html`

4. 页面打开后，顶部 **Root Path** 填：

```text
/data/workspace/amer-fpp-a
```

---

## 第三步：Health Check（两端口问题）

网页在 **18081**，API 在 **18082**。Knot 预览通常一次只暴露一个端口。

| 现象 | 处理 |
|------|------|
| 页面能开，Health Check 失败 | 再为 **18082** 生成一条预览链接 → 填到页顶 **Backend URL** |
| 两个预览同域不同端口 | 把 18082 预览填 Backend URL |
| 只有一个预览 | 暂时只用 **Chat 入口**（见下） |

在 `bot-gateway/.env` 可预填（生成新预览后替换）：

```bash
PUBLIC_WEB_BASE=https://你的18081预览域名
DEFAULT_API_BASE=https://你的18082预览域名
```

改完后：`bash knot-chat/restart_web.sh`

---

## 第四步：验收

```bash
# Knot 终端
curl -s http://127.0.0.1:18082/api/health
curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:18081/upload-docs.html
```

Mac 浏览器：
1. 打开 `https://xxx/upload-docs.html` → 能看到页面
2. 点 **Health Check** → ok
3. 点 US 演示按钮或上传测试 → 有下载
