## 1. “真正能用到和落实”的工作场景（非虚假AI界面）

以下每个场景都对应了**真实按钮/脚本/输出文件**，可直接执行。

### 场景A：提单前材料产出（高频）
- 触发方式：网页`文档产出`页 -> `生成提单文本` / `生成HC校验`。
- 后端接口：`/api/upload-and-generate`。
- 对应脚本：`generate_fpp_submission_blocks.py`、`check_hc_analysis.py`。
- 产物：`FPP提单文本块*.md`、`HC校验报告*.md`。
- 解决问题：减少手工拼装文本与基础校验重复劳动。

### 场景B：成本中心自动分摊（痛点1，High）
- 触发方式：网页`成本中心预检`页 -> `自动分摊生成HC` / `自动填入并执行（推荐）`。
- 后端接口：`/api/pain/build-allocation`。
- 对应脚本：`痛点攻坚实施包/build_hc_allocation.py`。
- 产物：`HC_Analysis_{US|CAN}_*.csv`。
- 解决问题：把“按供应商+人数+金额手工分摊”改为规则化生成，降低错算风险。

### 场景C：失效成本中心拦截（痛点3，Medium）
- 触发方式：网页`成本中心预检`页 -> `检查成本中心（上传状态表/默认状态表）`。
- 后端接口：`/api/pain/check-centers`。
- 对应脚本：`痛点攻坚实施包/check_cost_centers.py`。
- 产物：成本中心校验报告（含不可提单阻断清单）。
- 解决问题：提单前拦截失效中心，避免提交失败。

### 场景D：台账自动回填（痛点4，Low但影响时效）
- 触发方式：网页`台账回填`页 -> `台账自动更新`。
- 后端接口：`/api/pain/update-ledger`。
- 对应脚本：`痛点攻坚实施包/update_ledger.py`。
- 产物：`台账_自动更新_{US|CAN}_*.csv`。
- 解决问题：减少漏更、晚更、重复录入。

### 场景E：文档字段健康检查（历史特殊case）
- 触发方式：网页`文档产出`页 -> `文档字段健康检查（含特殊case）`。
- 后端接口：`/api/upload-and-generate`（`doc_health_check_upload`）。
- 对应脚本：`check_submission_doc_health.py`。
- 可识别：缺字段、状态异常、币种不一致、Invoice策略列偏移、可提交=N但无阻断原因等。
- 解决问题：把“经验型踩坑”前置成规则化拦截。

### 场景F：统计看板与月度复盘（新增提效）
- 触发方式：网页`文档产出`页 -> `统计分析（看板）` + `月度看板历史`。
- 后端接口：`/api/upload-and-generate`（`stats_analysis_upload`）。
- 对应脚本：`painpoint_stats_analyzer.py`、`import_real_monthly_history.py`。
- 当前看板能力：异常检测、稳定性监控、风险评分、时效分析、失败归因、国家综合统计、覆盖类型计数。
- 解决问题：把“感觉问题”升级为“量化问题”，指导下月优化优先级。

### 场景G：敏感字段脱敏上传（合规）
- 触发方式：网页`上传前脱敏（输出自动恢复）`勾选。
- 后端逻辑：上传CSV脱敏 -> 执行分析 -> 可选恢复输出文本。
- 解决问题：降低敏感信息暴露风险，同时保持结果可读。

### 场景H：批量与可运维能力（非技术操作者）
- 页面能力：`批量串行执行`、`执行历史`、`任务队列(暂停/继续/重试/取消/导出)`。
- 解决问题：多人协同下可追踪、可回放、可审计，不依赖“记忆式操作”。

## 4. 不是“虚假AI界面”的判定依据

- 有真实后端接口与本地脚本执行链路，不是前端静态演示。
- 每个按钮有可追踪输入、输出文件、执行日志和失败信息。
- 支持批量、历史、重试、导出，具备运维属性。
- 有可复核的数据产物（CSV/MD/JSON），不是只给一句“AI建议”。
- 关键痛点有对应自动化动作并已嵌入同一门户。
