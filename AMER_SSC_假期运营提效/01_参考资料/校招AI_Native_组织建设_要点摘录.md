# 校招 AI Native 组织建设 · 要点摘录（SSC 对标用）

> 来源：`校招团队AI Native组织建设探索与实践.pdf`  
> 用途：指导 FPP + 假期运营 **同一系统、多 Agent 演进**

---

## 核心判断

- **AI 组织 ≠ 用了 AI 的组织** — 要把工作流、数据、知识沉淀为 **组织级上下文**
- 校招当前：阶段 1→2，正在推阶段 3（MCP）
- SSC 可对标的演进：**工具辅助 → 信息聚合 → 上下文打通 → 智能协同**

---

## 四阶段路径

| 阶段 | 名称 | 校招 | SSC（FPP + Leave） |
|------|------|------|---------------------|
| 1 | 工具辅助 | WorkBuddy 个人提效 | Knot + Skills + Python 脚本 |
| 2 | 信息聚合 | 校招 AI 操作台 | upload-docs + 假期 Tab + portal |
| 3 | 上下文打通 | 三类 MCP | SSC 过程 MCP（比对报告、映射表、SOP） |
| 4 | 智能协同 | Multi-Agent | FPP Agent + Leave Agent + 合成 |

---

## 组织自检 8 指标（节选 SSC 适用）

1. **决策证据来源** — 策略能否追溯到数据、报告、SOP  
2. **工作流 Pipeline 化** — 导出→比对→报告→导入 是否固定  
3. **上下文完整度** — 同一员工能否串 WD、ADP、Prepayroll、处理备注  
4. **AI 建议采纳率** — 异常建议是否记录、复盘  
5. **数据沉淀质量** — 映射表是否有字段、版本、权限  
6. **决策响应速度** — 阻断出现后多久进入看板  
7. **人机协作方式** — AI 先整理，人做判断  
8. **组织学习速度** — case 能否跨月复用  

---

## 三类 MCP（并列，不互相替代）

| MCP | 回答的问题 | SSC 对标 |
|-----|------------|----------|
| 标准业务数据 MCP | 系统里发生了什么 | FPP/Payroll/WD/ADP 只读 |
| 流程/关系 MCP | 线索/任务如何经营 | Prepayroll checklist、提单状态 |
| 过程上下文 MCP | 过程中形成什么可复用知识 | **比对报告、映射表 changelog、异常处理** |

---

## Multi-Agent 三种协作模式

1. **广播订阅型** — 新比对报告入库 → Leave Agent 自动摘要阻断  
2. **圆桌会议型** — 月结前「SSC AI 晨会」：FPP + Leave + 合成  
3. **即问即答型** — 「本期 US Prepayroll 风险在哪」→ 并发查 MCP  

**前提**：没有统一 MCP，Multi-Agent 只能各说各话。

---

## 接入顺序（照抄）

```
样板闭环：报告 → MCP 入库 → 操作台/WorkBuddy 消费 → 展示
    ↓
扩展数据源（FPP 报告、Leave 报告、映射表）
    ↓
Multi-Agent（晨会、合成）
```

## 对「两个业务一个系统」的直接启示

| 校招做法 | SSC 落地 |
|----------|----------|
| 一个操作台多个 Tab | FPP Tab + 假期 Tab + 看板 |
| WorkBuddy 总入口 | Knot 先域路由，后期拆专科 Agent |
| 脚本/API 算数 | run_pain* + run_leave* |
| Agent 不做金额/天数计算 | 与 FPP guardrail 相同 |
