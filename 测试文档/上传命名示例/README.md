# 上传文件严格命名规范

上传到 Knot 工作区 **`上传输入/`** 时，请按下列格式命名（`YYYYMM` = 账单月，如 `202607`）。

## 格式

```
IN_{国家}_P{环节}_{说明}_{YYYYMM}.csv
```

| 前缀 | 含义 |
|------|------|
| `TPL_` | 空模板（只读参考，勿改） |
| `DEMO_` | 演示样例（培训用，带 `_202605`） |
| `IN_` | 同事实际上传文件 |

| 国家 | 代码 |
|------|------|
| 美国 | `US` |
| 加拿大 | `CAN` |

## 各环节对照

| 步骤 | 环节 | US 上传命名示例 | CAN 上传命名示例 |
|------|------|-----------------|------------------|
| 2 自动分摊 | P1 人数输入 | `IN_US_P1_人数_202607.csv` | `IN_CAN_P1_人数_202607.csv` |
| 4 提单文本 | P2 提单预填表 | `IN_US_P2_提单预填表_202607.csv` | `IN_CAN_P2_提单预填表_202607.csv` |
| 5 成本中心检查 | P3 状态表 | `IN_US_P3_成本中心状态表_202607.csv` | `IN_CAN_P3_成本中心状态表_202607.csv` |
| 6 台账更新 | P4 台账 | `IN_US_P4_台账_202607.csv` | `IN_CAN_P4_台账_202607.csv` |

步骤 3（HC 校验）通常直接用步骤 2 产出：`上传输出/HC_Analysis_US_*.csv`，无需重新上传。

## 模板与演示路径

| 类型 | US | CAN |
|------|----|-----|
| 模板 | `测试文档/模板/US/TPL_US_P*.csv` | `测试文档/模板/CAN/TPL_CAN_P*.csv` |
| 演示 | `测试文档/闭环CSV/US/DEMO_US_P*_202605.csv` | `测试文档/闭环CSV/CAN/DEMO_CAN_P*_202605.csv` |
