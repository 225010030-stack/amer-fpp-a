## 3) 输出文件分别代表什么（讲解口径）

### 3.1 自动分摊生成HC
- 典型输出：`HC_Analysis_US_*.csv` / `HC_Analysis_CAN_*.csv`
- 含义：基于人数和参数自动分摊金额到成本中心，形成后续预检与提交基础数据。

### 3.2 文档产出（上传并生成）
- 典型输出：`FPP提单文本块_<国家>_*.md`
- 含义：可直接用于提单系统粘贴的文本块，减少重复手填。

### 3.3 检查成本中心
- 典型输出：`成本中心有效性检查报告_<国家>_*.md`
- 含义：识别可用/失效/阻断成本中心，提前暴露无法提单风险。

### 3.4 台账自动更新
- 典型输出：`台账_自动更新_<国家>_*.csv`
- 含义：自动完成台账回填和去重更新，降低漏更、晚更、重复录入。

## 4) 其他按钮功能补充（讲解可选）

### 4.1 顶部连接区
- `保存`：保存 Backend URL 和 Root 到浏览器本地。
- `健康检查`：请求 `/api/health`，确认后端在线。

### 4.2 成本中心预检页（模板管理）
- `保存默认模板`：保存期间、主体、供应商、费用类型、金额、币种。
- `读取默认模板`：快速回填，减少重复输入。
- `重置默认模板`：清理本机默认配置。
- `导出模板JSON`：跨电脑迁移参数。
- `导入模板JSON`：恢复参数并做字段校验。
- `自动填入并执行（推荐）`：自动加载模板后直接跑分摊。

### 4.3 成本中心检查的两个模式
- `检查成本中心（上传状态表）`：适合临时替换最新状态表。
- `检查成本中心（用默认状态表）`：走系统默认状态表路径，操作更快。

### 4.4 执行历史页
- `查询历史`：按国家、步骤、时间筛选执行记录。
- `刷新运营看板`：查看今日/本周处理量、成功率、失败率、Top3 失败原因等。

### 4.5 任务队列页
- `加入队列`：批量把文件入队。
- `开始执行`：串行执行队列。
- `暂停/继续`：控制执行节奏。
- `失败重试`：重跑失败项。
- `导出队列记录`：导出追踪报表。
- 单条`取消`：可取消 queued/failed 项。

## 6) 现场常见报错与一句话处理

- 提示：`执行失败（服务错误）`  
  优先检查 Root 是否为 `/app`，再做健康检查。

- 提示：`Path is outside workspace root`  
  Root 填错，改为 `/app`（Docker）或后端允许的工作目录。

- 提示：`Please select CSV file`  
  上传了非 CSV 或未选文件，按本清单重选。

- 有健康检查但按钮失败  
  多数是字段不完整或上传错文件，先看 Raw JSON 的 `detail`。
