# KnotBot 配置中心（按配置界面分层）

> 用途：把 KnotBot 的配置动作按“界面分类”沉淀，方便后续迭代和交接。  
> 维护原则：每次改配置，都在对应分类目录补一份“变更记录”。

## 目录结构

- `01-智能体开发`
  - Prompt、Rules、Skills、Client 工具的基础配置
- `02-智能体集成`
  - 01 使用企微机器人
  - 02 API_SDK 调用
  - 03 发布为网页URL
  - 04 嵌入至Web网页
  - 05 Knot_CLI
  - 06 在流水线中使用
  - 07 定时运行
  - 08 对话结果同步企微
- `03-智能体运营`
  - 指令监控、执行统计、失败复盘
- `04-权限配置`
  - 可管理/可使用人员清单与审批记录
- `05-运行工作区与设备`
  - 运行主机、端口、进程、重启方法

## 迭代规则

1. 每次改动先改对应分类文件，不要只口头同步。  
2. 每次上线补一条记录：时间、改动人、改动项、回滚方式。  
3. 所有可执行命令都保留“可复制版本”，避免同事手敲。
