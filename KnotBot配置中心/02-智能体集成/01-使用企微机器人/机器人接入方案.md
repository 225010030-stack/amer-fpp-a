# 机器人接入方案（低 Token、可立即落地）

> 目标：让同事在机器人里发指令，就能触发本地/服务器脚本，结果以文件形式返回。  
> 适用场景：你当前的 KnotBot 配置界面（智能体开发）+ 现有 `工作提效` 脚本体系。

---

## 1. 总体架构（方案A）

机器人消息（如“目录预检”）  
-> Bot 命中 `Client 工具` 调用后端接口  
-> 后端执行已有脚本（`.py` / `.command`）  
-> 结果写入文件（`*.md`）  
-> Bot 回传“结果摘要 + 文件路径”

核心原则：
- 不靠大模型做计算，模型只做“路由和说明”
- 计算与检查全部由脚本完成（几乎不耗 token）
- 所有输出都落文件，便于审计和交接

---

## 2. 在你当前机器人界面的配置建议

基于你截图里的模块，建议这样配：

1) `Prompt`  
- 只保留路由规则，不让模型自由发挥。  
- 明确写：遇到关键词优先调用 Client 工具，不要直接回答。

2) `Client 工具`（关键）  
- 增加 3 个工具：
  - `run_precheck` -> 对应 `skill-01`
  - `run_hc_check` -> 对应 `skill-02`
  - `run_full_check` -> 一次跑两个检查

3) `Skills`  
- 保留已做的 `skill-01-directory-precheck` / `skill-02-hc-analysis-check` 作为规则说明来源。  
- 真正执行由 Client 工具后端完成（更稳定）。

4) `Rules`  
- 增加固定输出规则：  
  - 必须返回：状态、执行时间、输出文件路径  
  - 出错时必须返回：错误类型、可重试动作

5) `模型设置`  
- 选低成本模型即可，因为主要只是解析指令。

---

## 3. 指令设计（给同事直接用）

建议固定 4 条：

- `目录预检`
  - 行为：执行 `run_precheck`
- `HC校验`
  - 行为：执行 `run_hc_check`
- `总检查`
  - 行为：执行 `run_full_check`
- `生成提单文本块`
  - 行为：执行 `generate_fpp_submission_blocks.py`

可扩展参数（可选）：
- `月份=202606`
- `文件=HC_Analysis_统一模板.csv`

---

## 4. 回调格式（Bot <- 后端）

统一 JSON 响应（示例）：

```json
{
  "ok": true,
  "action": "run_full_check",
  "started_at": "2026-06-08T14:15:00+08:00",
  "elapsed_ms": 1432,
  "summary": "目录预检与HC校验已完成",
  "outputs": [
    "/Users/zhangwenjing/Desktop/工作提效/目录预检输出/目录预检报告_20260608_141500.md",
    "/Users/zhangwenjing/Desktop/工作提效/目录预检输出/待修复清单_20260608_141500.md",
    "/Users/zhangwenjing/Desktop/工作提效/HC校验输出/HC校验报告_20260608_141501.md",
    "/Users/zhangwenjing/Desktop/工作提效/HC校验输出/HC待修复清单_20260608_141501.md"
  ]
}
```

失败格式（示例）：

```json
{
  "ok": false,
  "action": "run_hc_check",
  "error_code": "SCRIPT_NOT_FOUND",
  "message": "Missing check_hc_analysis.py",
  "suggestion": "确认工作提效目录完整后重试"
}
```

---

## 5. 错误处理规范（必须加）

常见错误与处理：
- `SCRIPT_NOT_FOUND`：脚本不存在 -> 提示检查部署目录
- `INVALID_ARGUMENT`：参数错误 -> 回传正确命令示例
- `PERMISSION_DENIED`：无执行权限 -> 自动尝试 `chmod +x`
- `RUNTIME_FAILED`：脚本执行失败 -> 把 stderr 写入 `错误日志_*.md`
- `TIMEOUT`：超时 -> 终止进程并提示拆分任务

所有错误都要落地到文件：
- `工作提效/运行日志/错误日志_YYYYMMDD_HHMMSS.md`

## 6. 后端执行映射（建议）

- `run_precheck`
  - `python3 .cursor/skills/skill-01-directory-precheck/scripts/run_precheck.py --root .`
- `run_hc_check`
  - `python3 .cursor/skills/skill-02-hc-analysis-check/scripts/run_hc_check.py --root .`
- `run_full_check`
  - `./一键运行_总检查_skill01+02.command`
- `generate_submission_blocks`
  - `python3 ./generate_fpp_submission_blocks.py --input ./提单预填表模板.csv --output ./FPP提单文本块.md`

---

## 7. 上线顺序（推荐）

第1阶段（当天）：
- 先只接 `总检查` 一个指令，打通完整链路。

第2阶段（1-2天）：
- 加 `目录预检`、`HC校验`、`生成提单文本块`。

第3阶段（稳定后）：
- 增加权限控制（谁能触发、谁能看输出）。

---

## 8. 同事使用文案（可直接发群）

可用指令：
- `总检查`
- `目录预检`
- `HC校验`
- `生成提单文本块`

收到回复后，打开返回的文件路径查看结果并按清单处理即可。  
不需要终端，不需要 token。
