# KnotBot 控制台逐屏配置步骤（最后一公里）

> 目标：把机器人消息接到 `bot-gateway`，实现“发指令 -> 跑脚本 -> 返回文件路径”。  
> 适用：你当前的 KnotBot「智能体开发」页面。

---

## 0. 开始前准备（先确认）

- 后端已启动（你现在是 `18082`）：
  - 健康检查：`http://127.0.0.1:18082/api/health`
- 已有可执行动作：
  - `run_precheck`
  - `run_hc_check`
  - `run_full_check`
  - `generate_submission_blocks`
- 参考文件：
  - `bot-gateway/config/command-map.json`
  - `机器人webhook适配器_配置清单.md`

---

## 1. 进入「智能体开发」页（你截图的这个页）

左侧区域：
- 智能体名称：可填 `Anna 小助手`
- 工作内容短效：填一句“触发脚本执行并返回文件路径”

Prompt 建议直接填（可复制）：

```text
你是流程路由助手，不做复杂推理。
遇到以下关键词，必须调用 Client 工具触发执行，不直接自由回答：
1) 总检查 -> run_full_check
2) 目录预检 -> run_precheck
3) HC校验 -> run_hc_check
4) 生成提单文本块 -> generate_submission_blocks

输出格式固定：
- 状态（成功/失败）
- 动作名
- 输出文件路径列表
- 下一步建议
```

---

## 2. 配置「Client 工具」（关键）

在中间列表点击 `Client 工具 -> 添加`

新增 1 个 HTTP 工具（建议名：`fpp_gateway_webhook`）：

- 请求方法：`POST`
- URL：`http://127.0.0.1:18082/api/webhook/knotbot`
- Header：
  - `Content-Type: application/json`
  - `X-Bot-Token: <你的token>`（若你启用了 token 校验）
- Body 模板：

```json
{
  "text": "{{user_input}}"
}
```

说明：
- `{{user_input}}` 换成 KnotBot 实际变量名（如平台里叫 `{{message}}` 就替换）

---

## 3. 配置「Rules」（降低跑偏）

点击 `Rules -> 添加`，增加 3 条：

1) **优先工具调用规则**  
- 识别到关键词必须走 `Client 工具`

2) **固定输出规则**  
- 回复中必须包含：`状态`、`动作`、`文件路径`

3) **失败回退规则**  
- 调用失败时输出：
  - 错误原因
  - 可重试指令（例如“请重试：总检查”）

---

## 4. 配置「Skills」（可选但建议保留）

你已做了：
- `skill-01-directory-precheck`
- `skill-02-hc-analysis-check`

这里建议保留启用，用于增强机器人解释能力。  
真正执行仍由 Client 工具 + 网关完成（更稳、更省 token）。

---

## 5. 配置「模型设置」

- 选择低成本模型（因为只做路由）
- 不需要高推理档位

---

## 6. 对话测试（右侧测试窗）

按顺序输入并验证：

1) `总检查`
- 预期：返回 `run_full_check` 成功 + 文件路径

2) `目录预检`
- 预期：返回 `目录预检输出/*.md` 路径

3) `HC校验`
- 预期：返回 `HC校验输出/*.md` 路径

4) `生成提单文本块`
- 预期：返回 `FPP提单文本块.md` 路径

## 8. 常见问题（快速处理）

## Q1: 返回 401
- 原因：`X-Bot-Token` 不匹配
- 处理：核对网关环境变量和控制台 Header

## Q2: 返回“未识别指令”
- 原因：关键词不在 `command-map.json`
- 处理：补映射后重启网关

## Q3: 返回执行失败
- 原因：脚本权限或路径问题
- 处理：查看返回里的 `log_file`，按日志修复

## Q4: 机器人回复慢
- 原因：执行动作本身耗时
- 处理：先返回“处理中”，再回调结果（后续可做异步）
