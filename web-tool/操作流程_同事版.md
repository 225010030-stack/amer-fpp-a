# 操作流程（操作者版）

## 一、首次使用
1. 双击 `一键重启并打开上传文档页.command`
2. 页面右上角选择语言（中文/English）
3. 在连接区确认：
   - Backend URL（默认 `http://127.0.0.1:18082`）
   - Root Path（自动识别项目目录）
4. 点击 `Health Check`

## 二、按国家隔离操作

页面分成两个完全独立区域：
- US Workspace：只做美国流程文件
- CAN Workspace：只做加拿大流程文件

请不要混用区域（例如 US 文件放到 CAN 区域）。

## 三、基础产出操作

### 1) 生成提单文本
- 上传：`提单预填表模板.csv`
- 点击：`生成提单文本 / Submission Blocks`

### 2) 生成HC校验
- 上传：`HC_Analysis_统一模板.csv`
- 点击：`生成HC校验 / HC Check`

## 四、自动化动作

### 1) 自动分摊
- 上传人数输入CSV
- 填写期间、主体、供应商、金额等参数
- 点击：`自动分摊 / Build Allocation`

### 2) 成本中心检查
- 上传 source 文件（待提单源）
- 上传 status 文件（成本中心状态表）
- 点击：`成本中心检查 / Check Centers`

### 3) 台账自动更新
- 上传 prefill 文件（必填）
- 上传 ledger 文件（可选）
- 点击：`台账自动更新 / Update Ledger`

## 五、结果处理
1. 在当前国家区域的 Result 查看执行结果
2. 点击输出链接下载文件
3. 建议将产物归档到当月文件夹

## 六、常见问题

- `Failed to fetch`：后端未启动，重新运行一键启动脚本。
- 无输出链接：检查上传文件是否为空或字段不匹配模板。
- 端口占用：先关闭旧的服务进程后重启。
