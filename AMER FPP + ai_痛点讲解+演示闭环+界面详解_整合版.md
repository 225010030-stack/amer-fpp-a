# 第一部分：痛点讲解（开场 5-8 分钟）

## 1.1 总体闭环框架

四大痛点不是四个独立问题，而是一条链路：

`输入标准化 → 自动分摊 → 提单产出 → 阻断拦截 → 台账回填 → 复盘固化`

对应 SOP 环节：

| SOP环节 | 内容 | 对应痛点 |
|---------|------|----------|
| 环节1 Source | 材料收集与标准化 | 输入质量 |
| 环节2 Process | 分摊计算与校验 | 痛点1 |
| 环节3 Submit | 提单字段产出与录入 | 痛点2 |
| 环节4 Exception | 失效中心阻断处理 | 痛点3 |
| 环节5 Output | 台账回填与归档 | 痛点4 |
| 环节6 Improve | 复盘、指标回看、规则更新 | 持续优化 |

---

## 1.2 痛点1：分摊耗时易错（High，约17单/月）

**发生位置**：环节2 分摊计算与校验  
**以前怎么做**：人工 Excel 分摊，供应商多、数据量大，总额不平、期间/币种不一致  
**现在怎么解决**：`成本中心预检 → 自动分摊生成HC`  
**输入**：人数 CSV + 页面参数（Period/Country/Entity/Vendor/Fee Type/Invoice Total/Currency）  
**输出**：`HC_Analysis_{US|CAN}_*.csv`  
**验收标准**：分摊汇总 = 账单总额；期间/币种一致  
**责任人**：主责操作者；复核财务/流程负责人

---

## 1.3 痛点2：提单录入重复（High，约20单/月）

**发生位置**：环节3 FPP提单与审批链  
**以前怎么做**：同类单据重复手填大量字段，易漏填、错填  
**现在怎么解决**：`文档产出 → 生成提单文本` / `生成HC校验` / `文档字段健康检查`  
**输入**：`提单预填表模板.csv`、`供应商字段字典模板.csv`  
**输出**：`FPP提单文本块_*.md`、`HC校验报告_*.md`、`提单文档字段健康检查报告_*.md`  
**验收标准**：字段完整、状态与币种一致、特殊 case 已识别  
**责任人**：主责操作者；复核业务接口人/流程负责人

---

## 1.4 痛点3：失效成本中心阻断（Medium）

**发生位置**：环节4 异常处理（提交前阻断点）  
**以前怎么做**：提单时报失效中心，流程中断，主体无法提单  
**现在怎么解决**：`成本中心预检 → 检查成本中心（上传状态表）`  
**输入**：Source（HC Analysis）+ Status（成本中心状态表）  
**输出**：`成本中心有效性检查报告_*.md`（含阻断清单）  
**验收标准**：无效/停用中心提前识别；可替代则替换留痕，不可替代则升级主数据  
**责任人**：主责操作者；配合主数据维护人

---

## 1.5 痛点4：台账漏更/晚更/重复（Low）

**发生位置**：环节5 台账回填与归档  
**以前怎么做**：提交后未及时回填，或重复写入，影响提单时效  
**现在怎么解决**：`台账回填 → 台账自动更新`  
**输入**：Prefill（提单预填表）+ Ledger（台账模板/实际台账）  
**输出**：`台账_自动更新_{US|CAN}_*.csv`  
**验收标准**：关键字段去重；更新时间与责任人完整  
**责任人**：主责操作者；配合台账管理员

---

## 1.6 培训统一口径（可直接念）

“这4个痛点不是分别优化，而是一个链路闭环。每一步都必须有输入文件、输出文件和责任人，确保可追踪、可复盘、可交接。工具不是黑盒，所有结果都可在报告文件与原始 JSON 中复核。”

# 第三部分：每个界面设置细节、功能与作用

## 3.1 页面总览

页面分为：**顶部连接区**、**左侧导航**、**右侧业务区**、**结果区**、**操作文档区**。  
US 与 CAN 严格分离：按钮、模板、结果、进度互不影响。

---

## 3.2 顶部连接区（全局，所有页面共用）

| 元素 | 设置/操作 | 功能与作用 |
|------|-----------|------------|
| Backend URL | 填后端地址 | Docker 本地：`http://127.0.0.1:18082`；内网部署：`https://你的域名` |
| Root Path | 填工作根目录 | Docker 必须填 `/app`；填错会报 `Path is outside workspace root` |
| 保存 | 点击 | 将 Backend + Root 存到浏览器 localStorage，下次自动带入 |
| 健康检查 | 点击 | 请求 `/api/health`，确认后端在线；演示/操作前必做 |
| 面包屑 | 自动显示 | 如“当前：US / 成本中心预检”，防止迷路 |
| 语言切换 | 中/英 | 导航、提示、摘要同步切换 |
| 安全说明（绿条） | 固定显示 | 强调本地运行、文件本机处理、不上传外网 |

---

## 3.4 页面一：文档产出

**页面作用**：把 CSV 转成可提交文档，不修改台账；适合提交前检查和材料准备。

| 设置/按钮 | 输入 | 输出 | 功能说明 |
|-----------|------|------|----------|
| 上传前脱敏（勾选） | — | 输出自动恢复 | 敏感字段脱敏上传，结果可恢复，适合演示/外发 |
| 生成提单文本 | 提单预填 CSV | `FPP提单文本块_*.md` | 解决痛点2：减少重复手填 |
| 生成HC校验 | HC Analysis CSV | `HC校验报告_*.md` | 分摊结果校验报告 |
| 文档字段健康检查 | 提单预填 CSV | 健康检查报告 | 缺字段、阻断项、历史特殊 case |
| 统计分析（看板） | 提单预填 CSV | 统计 JSON + 看板摘要 | 风险、异常、时效、失败归因 |
| 批量串行执行 | 多选 CSV + 动作下拉 | 逐个产出 | 月初多供应商批量处理 |

---

## 3.5 页面二：成本中心预检

**页面作用**：提交前识别失效/未维护成本中心，自动分摊 HC，避免提单失败。

### 参数区（减少手填）

| 字段 | 作用 | 设置建议 |
|------|------|----------|
| Period | 账期 | 下拉或手选，如 202605 |
| Country | 国家 | US 固定 US；CAN 固定 CAN |
| Entity | 主体 | US：Tencent America LLC 等；CAN：Tencent Canada |
| Vendor | 供应商 | 下拉选择，联动费用类型 |
| Fee Type | 费用类型 | 与供应商匹配 |
| Invoice Total | 账单总额 | 可点“上次金额快捷标签”减少手填 |
| Currency | 币种 | US=USD，CAN=CAD |
| 供应商预设标签 | 一键填 Vendor/Fee/Entity/Currency | 如 Cigna 预设、SunLife 预设 |

### 模板管理按钮

| 按钮 | 作用 |
|------|------|
| 保存默认模板 | 把当前参数存本机，下次一键读取 |
| 读取默认模板 | 回填已保存参数 |
| 重置默认模板 | 清空该国默认配置 |
| 导出模板JSON | 跨电脑迁移参数 |
| 导入模板JSON | 导入并校验字段完整性 |

### 执行按钮

| 按钮 | 输入 | 输出 | 对应痛点 |
|------|------|------|----------|
| 自动分摊生成HC | 人数 CSV + 参数 | `HC_Analysis_*.csv` | 痛点1 |
| 自动填入并执行（推荐） | 人数 CSV | 同上 | 自动读模板后执行 |
| 检查成本中心（上传状态表） | source + status CSV | 有效性检查报告 | 痛点3 |
| 检查成本中心（用默认状态表） | 仅 source CSV | 同上 | 更快，用系统默认状态表 |

## 3.8 页面五：执行历史

**页面作用**：留痕、复盘、管理看数。

### 1页运营看板（L1 + L2）

| 指标 | 含义 |
|------|------|
| 今日/本周处理单量 | 按 started_at 统计 |
| 成功率 / 失败率 | 本周成功/失败占比 |
| 各步骤平均耗时 | output / center / ledger |
| 失败原因 Top3 | 字段/文件/服务分类 |
| US vs CAN 对比 | 处理量、成功率 |
| 高风险单占比 / 漏更率 / 阻断处理时长 | 管理视角（基于日志推导） |

### 执行历史筛选

| 设置 | 作用 |
|------|------|
| 步骤下拉 | 按 output/center/ledger 等筛选 |
| 开始/结束时间 | 时间区间过滤 |
| 查询历史 | 默认最近 50 条，含时间、国家、步骤、动作、状态、耗时 |

# 第四部分：常见报错与处理

| 提示 | 原因 | 处理 |
|------|------|------|
| 执行失败（服务错误） | Root 错误或后端异常 | Root 改为 `/app`，再健康检查 |
| Path is outside workspace root | Root 填了本机路径而非容器路径 | Docker 场景必须用 `/app` |
| Please select CSV file | 未选文件或格式不对 | 按清单重选 CSV |
| 审批中/仓库空 | iGate 策略未通过 | 补申请 `git.woa.com`，等通过后同步仓库 |
