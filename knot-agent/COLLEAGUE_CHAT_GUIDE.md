# 同事 Knot Chat 使用指南（四痛点 · 零幻觉）

在 Knot chat 里直接发下面这些话即可，**不用记脚本路径**。

## 第一次用

发：**`帮助`** 或 **`US演示`**

- `帮助` → 列出全部指令
- `US演示` → 用内置样例 CSV 跑通四痛点闭环（推荐培训/demo）

## 四痛点指令

| 发这句话 | 做什么 | 输出在哪 |
|---------|--------|---------|
| **自动分摊** / **痛点1** | 人数 CSV → HC Analysis | `上传输出/HC_Analysis_US_*.csv` |
| **生成提单文本块** / **痛点2** | 预填表 → 提单文本 | `上传输出/FPP提单文本块_*.md` |
| **检查成本中心** / **痛点3** | 源文件+状态表 → 阻断报告 | `上传输出/成本中心有效性检查报告_*.md` |
| **台账更新** / **痛点4** | 预填表+台账 → 更新台账 | `上传输出/台账_自动更新_*.csv` |

## 预检指令

| 发这句话 | 做什么 |
|---------|--------|
| **目录预检** | 检查 FPP 目录结构 |
| **HC校验** | 校验 HC Analysis 格式 |
| **总检查** | 目录预检 + HC校验 |

## 用自己的文件

1. Knot 工作区 → **文件管理** → 上传到 `上传输入/`
2. chat 里说：**「用上传输入/xxx.csv 做分摊，期间 202606」**
3. 智能体会带路径执行脚本（不会编造结果）

## 怎么确认没幻觉

每条回复应包含：
- **【输出文件】** 一行 `DONE:` 开头的真实路径
- 可自己去 **文件管理** 打开该文件复核

若智能体只聊天、没有执行命令 → 重新发指令或联系管理员检查 Prompt/Client 配置。
