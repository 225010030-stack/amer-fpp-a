# 同事入口方案：网页 vs 聊天（推荐）

## 结论（推荐混合，但以聊天为主）

| 入口 | 适合谁 | 优势 | 局限 |
|------|--------|------|------|
| **企微 / Knot 聊天框** ⭐主入口 | 全体操作同事 | 零安装、像 HR 助手点选、可推送到企微 | 多文件上传不如网页直观 |
| **网页 upload-docs.html** ⭐补充 | 需上传多 CSV + 填表单 | 四按钮、拖拽上传、参数表单 | 需记 URL、需后端启动 |

**推荐落地：**
1. **日常**：同事在 **企微智能机器人** 或 **Knot 网页 URL** 里聊天
2. **发「主菜单」**：出现 SOP 分步引导（类似你截图的 HR 助手）
3. **选数字/关键词**：助手说明要什么文件 → 执行脚本 → 返回 DONE 路径
4. **上传复杂批次**：助手回复「网页入口」→ 打开 upload-docs 四按钮

你已配置的 **企微智能机器人（已连接）** + **对话结果同步企微** 是正确的同事入口。

---

## 为什么聊天做主入口

1. 未来账单是 **重复性 SOP**，菜单比记 URL 更友好  
2. 你已做好 **防幻觉**（脚本 + DONE 路径），聊天只负责 **引导 + 路由**  
3. 企微是同事已有工具，无需培训新系统  
4. HR 助手同款体验：主菜单 → 选环节 → 提示文件 → 执行  

---

## 什么时候用网页

- 痛点1 要填 Period / Invoice Total 等多个表单字段  
- 一次上传 2～3 个 CSV（如痛点3 source + status）  
- 培训演示时投屏操作  

启动方式（工作区终端）：
```bash
bash /data/workspace/amer-fpp-a/knot-chat/start_web.sh
```

## Knot 集成页建议开启项

| 功能 | 建议 | 原因 |
|------|------|------|
| 使用企微智能机器人 | ✅ 已连接 | 同事主入口 |
| 发布为网页 URL | ✅ 开启 | 培训 / 无企微时备用 |
| 对话结果同步到企微 | ✅ 已开 | 留痕、转发 |
| 嵌入至 web 网页 | 可选 | 内部门户 iframe |
| 定时运行 | 关 | 账单是人工触发，非定时 |
