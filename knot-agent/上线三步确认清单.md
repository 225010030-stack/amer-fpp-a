# 上线三步确认清单（管理员）

完成下面 3 项后，同事可在企微/Knot 发 **主菜单** 或 **US演示**。

---

## ✅ 第 1 步：更新 Prompt 并发布（Knot 网页操作）

1. 打开 **Knot → 智能体开发** → 你的智能体（Anna）
2. 打开本地文件 **`knot-agent/PROMPT.md`**，**全选复制**
3. 粘贴到智能体 **Prompt** 框（覆盖旧内容）
4. 确认 **Client 工具** 已勾选：
   - ✅ 执行命令
   - ✅ 读取文件（建议）
5. 确认 **工作区设置** → 指定 `amer-fpp-a`
6. 点击右上角 **「发布更新」**

---

## ✅ 第 2 步：工作区拉最新代码（Knot 终端）

工作区 → **登录** 或终端，执行：

```bash
cd /data/workspace/amer-fpp-a
git pull origin main
chmod +x knot-chat/*.sh
ls knot-chat/run_sop_menu.sh
```

最后一行应能看到 `run_sop_menu.sh`。

---

## ✅ 第 3 步：chat 验收测试

在 **对话测试** 或 **企微机器人** 里依次发：

| 发送 | 预期 |
|------|------|
| `主菜单` | 出现 0-7 环节 + 每步需要什么文件 |
| `US演示` | 多个 `DONE: ...上传输出/...` |
| `2` | 说明分摊需要什么文件，然后执行或询问样例 |

---

## 同事入口（配置完成后告知同事）

- 企微：找 FPP 智能机器人
- 第一句话：**主菜单** 或 **US演示**
- 一页纸：`knot-agent/同事一句话上手.md`

---

## 若主菜单报错

| 报错 | 处理 |
|------|------|
| run_sop_menu.sh not found | 第 2 步 git pull |
| 只聊天不执行 | 第 1 步 Client + 发布 |
| 菜单内容不对 | Prompt 是否最新 PROMPT.md |
