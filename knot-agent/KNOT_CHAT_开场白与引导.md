# Knot Chat 功能引导配置（你的 chat 链接）

**Chat 入口：**  
https://knot.woa.com/chat?web_key=05b6ce9207b64b89ad64794a34a36368&close=0

**设计原则：**

| 渠道 | 交互方式 | 说明 |
|------|---------|------|
| **Knot chat / 企微** | 回复 **数字 0–7** 或关键词 | 模拟 HR 点选，Prompt + 开场白配置 |
| **chat-menu.html** | 真按钮点击 | 工作区预览/内网端口，执行同一套脚本 |
| **upload-docs.html** | 四按钮 + 拖文件 | 复杂 CSV 上传 |

Knot chat **不能**像 HR 机器人那样在消息里放原生蓝色回调按钮；用 **开场白 + 每条 footer + 数字路由** 达到同样效果。

---

## 第 1 步：配置开场白（智能体开发页）

找 **「开场白」/「欢迎语」/「首次对话」** 字段（名称因版本而异），粘贴：

文件：`knot-agent/WELCOME_粘贴到Knot开场白.txt`

若无单独开场白字段 → 靠 Prompt 第一节（用户发「你好」时触发主菜单）。

---

## 第 2 步：更新 Prompt

1. 打开 `knot-agent/PROMPT.md` **全文复制**
2. 粘贴到 **智能体开发 → Prompt**
3. **发布更新**

Prompt 已规定：
- 首次 / 「你好」/ 「帮助」→ 执行 `run_agent.sh 主菜单`
- 用户发 **0–7** → SOP 说明 → 执行脚本
- **每条回复末尾** 加 footer：

```text
---
有帮助 | 无帮助 | 需要人工支持 | 主菜单 | US演示
```

---

## 第 3 步：Rules（建议加 2 条）

在 **Rules** 里添加：

**规则 1 · 数字路由**  
用户只发单个数字 0–7 时，必须按 SOP 表路由到对应 `run_agent.sh` 指令，不得自由发挥。

**规则 2 · 固定 footer**  
每条回复最后一行必须是：  
`有帮助 | 无帮助 | 需要人工支持 | 主菜单 | US演示`

---

## 第 4 步：Client 工具

勾选：
- ✅ **执行命令**
- ✅ **读取文件**

---

## 第 5 步：同事怎么用（发这个链接）

把 chat 链接发给同事，第一句话教他们发：

```text
主菜单
```

或

```text
0
```

（US 演示）

## chat-menu 网页（真点击，可选）

- 工作区内：`http://127.0.0.1:18081/chat-menu.html`（curl 200 已 OK）
- 同事 Mac 浏览器 **不能直接** 开 127.0.0.1 → 需 Knot **启动并预览** 端口 18081
- 在 Prompt/开场白里 **不必** 写 127.0.0.1；对外只推 **chat 链接 + 发数字**

若以后有内网域名映射到 18081，再在开场白加一行「点击菜单：http://xxx/chat-menu.html」。

---

## 第 6 步：验收（在 chat 链接里测）

打开你的 chat URL，依次发：

1. `你好` → 应出现主菜单 + footer  
2. `0` → US演示 + 多个 DONE  
3. `2` → 分摊说明 + 执行 + DONE  
4. `主菜单` → 再次显示 0–7  

---

## 企微智能机器人

已连接企微时，同事在企微里发同样数字即可；与 Knot 网页 chat **同一套 Prompt**。

在企微机器人 **详细配置 → 欢迎语** 可粘贴与 `WELCOME_粘贴到Knot开场白.txt` 相同内容。
