# AMER FPP · 0–7 测试指令文档（US / CAN）

> 工作区：`/data/workspace/amer-fpp-a`  
> 输出根目录：`上传输出/`、`HC校验输出/`、`目录预检输出/`  
> 成功标准：回复或终端里有 **`DONE: /data/workspace/amer-fpp-a/...`** 路径

---

## 测试前准备

1. **Knot 工作区终端**（不是本机 Mac）：

```bash
cd /data/workspace/amer-fpp-a
git pull origin main
chmod +x knot-chat/*.sh
```

2. **上传自有文件时**：Knot → 工作区 → **文件管理** → 上传到 `上传输入/`，严格命名 `IN_{US|CAN}_P{环节}_{说明}_{YYYYMM}.csv`（见 `测试文档/上传命名示例/README.md`）。

3. **账单月**：除 `0` 演示外，必须写 **6 位 YYYYMM**（如 `202607`），系统不默认月份。

4. **先选国家**：chat 先发 `主菜单` → `US主菜单` 或 `CAN主菜单`，或命令带前缀 `US …` / `CAN …`。

# 怎么打开输出文件

1. 复制回复里的 **`DONE:`** 完整路径  
2. Knot → 工作区 → **文件管理** → 进入对应文件夹  
3. 文件名里 **`20260709_143022`** = 运行时间；**账单月** 以你命令里的 `202607` 为准  

| 输出类型 | 文件夹 |
|----------|--------|
| 分摊 HC、提单文本、成本中心报告、台账 | `上传输出/` |
| HC 校验报告 | `HC校验输出/` |
| 目录预检报告 | `目录预检输出/` |

# 常见失败

| 现象 | 原因 | 处理 |
|------|------|------|
| 必须指定 YYYYMM | 没写账单月 | 命令加 `202607` |
| 必须指定 US/CAN | 没锁国家 | 先 `US主菜单` 或命令带 `US` |
| Input not found | 路径错或没上传 | 检查 `上传输入/` 文件名 |
| CAN 目录报错混在 US 预检里 | 没加国家 | 用 `目录预检 202607 US` |
| chat 无 DONE | 只聊天未执行脚本 | 检查 Client「执行命令」+ Prompt 已发布 |
